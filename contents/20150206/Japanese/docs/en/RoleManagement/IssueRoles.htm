Issue Role(s)

You can issue Role(s) to:

·           Cell users

·           Other cells

Cell Users

You can issue Role(s) to users (accounts) residing within the Cell itself. To register a Role, see Role Create API. To link this Role with the particular cell, you need to link it with the respective Cell. See Role Link Create API.

Other Cells

You can issue Role(s) to the users (accounts) of other cells using the following three Cell Control Objects:

·           External cell

·           Relation

·           External role

External Cell

External Cell refers to cells outside a particular cell. An External Cell can access the particular cell based on the Role assigned to it.

Figure 1 shows how External Cells can access the main cell.

 

Figure 1: Other Cells - External Cell

To register the external cell with the particular cell:

1.     Create an external cell in CellA.

2.     Define the CellB URL as a parameter in CellA.

3.     Validate whether the specific user (account) of CellB is authenticated to access CellA. The authentication process uses the dc_target parameter to pass token value. This token value helps in validating the specific user (account).

For more information on how to create a $link with the Role, see Role Link APIs.

Based on the RBAC, User of CellB is allowed to access CellA using the Employee Role.

Relation

Relation refers to the relationship between cell and an external cell(s).

Figure 2 shows the relationship between cell and external cells.

 

Figure 2: Other Cells - Relation

 

To register multiple external cells with the particular cell:

1.     Create two or more external cells in CellA. Define the URL of CellA as a parameter in CellB and CellB. This parameter is set in the dc_target parameter.

2.     Create a Relation, such as EmpBenefit in CellA. Further, this relation is linked with the Role using $links.

3.     Validate whether the specific user (account) of CellB is authenticated to access CellA. The authentication process uses the dc_target parameter to pass token value. This token value helps in validating the specific user (account).

For more information on how to create a $link with the Role, see Role Link APIs.

Based on the RBAC, the user of CellB and CellC are allowed to access CellA using the Employee Role.

External Role

An External Role is registered within a cell that helps users from a cell to access the external cell. The external role is granted to a user of the cell to access the external cell through the access privileges of external role . This particular external cell can exist within the same environment or outside the environment.

To access the particular cell, you must map the external role with the role of the external cell. Also, you must define a relation between these two cells.

Figure 3 shows the relationship between other cells using External Role.

 

Figure 3: Other Cells - External Role

To register multiple external cell with the particular cell:

1.     Create two or more external cells in CellA. Define URL of CellA as a parameter in CellB and CellC. This parameter is set in the dc_target parameter.

2.     Create a Relation such as EmpBenefit in CellA. Further, this relation is linked with two or more external cells using $links.

3.     Register External Role, such as ExtRole in CellA and specify the address of Role2 in the parameters. This way you are associating External Role with Roles of CellB and CellC.

4.     Associate Relation with External Role.

5.     Then, define Role and associate Role with the Extrnal Role in CellA using $link.

For more information on how to create a $link with the Role, see Role Link APIs.

Based on the RBAC, the user of CellB and CellC are allowed to access CellA using the External Role (ExtRole).
