Create External Role

概要

This API registers the External Role. An external role is registered within a cell that helps users from a cell to access the details of other external cell.

必要な権限

auth

制限事項

·           'Content-Type' header of the request is treated as application/json

·           Request body accepts only json format

·           'Content-Type' header of the response only supports application/json. The response body appears in json format

·           If you specify the XML or atom as a $format query option, it is not an error, but there can be no assurance of the data availability/correctness in the response body

リクエストボディ

Header name

Summary

Valid value

Required

Remarks

ExtRole

External role name (URL)

Number of digits: 1-1024

According to the format of URI scheme：http/https/urn

When an ExtRole is attached to a Box: /{app name}/__role/__/{Role name}

However, if the Schema information is not registered in the Box, it is considered that the ExtRole is not attached to the Box.

When an ExtRole is not attached to a Box: /{Cell name]/__role/__/{Role name}

○

 

_Relation.Name

Relation of target relation name

Number of Digits: 1-128

Character type: alphanumeric characters, - (hyphen) , _ (underscore),  + (plus) and : (colon)

However, you cannot specify the first character as _ (underscore) and : (colon).

In Relation registration API, the registered Relation is null

○

 

_Relation._Box.Name

Box name Relation of relationship target brute string

Number of digits: 1-128

Character type: alphanumeric characters, - (hyphen) and _ (underscore).

In Relation registration API, the linked Box name of the registered Relation is null
