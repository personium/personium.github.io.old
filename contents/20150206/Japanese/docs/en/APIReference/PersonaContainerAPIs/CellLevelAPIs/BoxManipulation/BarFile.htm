Bar File

Bar file contains the Box data in a hierarchical format, which contains resources, such as Odata, WebDAV, and Service of Box. This data is archived in the ZIP file format.

Specifications

Bar file comes in a ZIP format. The ZIP64 format is also acceptable.

In addition, ZIP files encryption is not supported.

Directory Structure

The directory structure of the Bar file is as follows:

  • "★ required" implies that the specified directory is essential during the installation of the bar file. If the specified directory does not exist, an error "400 - Bad Request" is returned. It also shows an error, if the structure of the bar file is not created in the following order.

Bar File Version Control

bar file version will be upgraded when, data structure are changed because of some enhancements and at the back compatibility could not be ensured.

·           Version won’t be upgraded just because of adding files.

·           Versions should be upgraded for file name change and delete.

90_rootprops.xml

The PROPFIND method indicates the XML data for all the levels of the Box, which will be exported as bar file.

Please refer to the “GET Property” API for more details about the xml data.

The URL of the BOX to be installed, is described as "dcbox :/".

When installing bar file, the creation date in the <prop> and all the data except get last modified will be installed as shown in the following example.

·           resourcetype: Set the type of collection.

·           acl: Set the permissions

·           Other: Set by PROPPATCH

It shows the schema definition of user OData. Retrieved XML based $metadata of OData collection while exporting a bar file.

For details of the XML data, see Section schema_get.

contents/Service/{src.js}

It registers the Box under the destination to {Service}/__src/{src.js}, the source file that is stored in the bar/90_contents/{Service}/{src.js}.

·           Cannot run as service, when no collection {Service} is defined （PROPPATCH） in bar/00_meta/90_rootprops.xml.

·           Cannot entry {src.js}, when {Service}/__src is not defined in bar/00_meta/90_rootprops.xml

·           Set the Content-Type of {src.js} according to {Service}/__src/{src.js}, defined in bar/00_meta/90_rootprops.xml
