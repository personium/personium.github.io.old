XML Structure

The Body in the XML should be according to the following schema:

項目名

Namespace

Node Type

概要

備考

propertyupdate

D:

Elements

Represents the root of propertyupdate, remove and set tags are its children nodes that resides in it

 

set

D:

Elements

Represents a property setting, which has at least one prop child

Lists the property values to be set for a resource. The 'set' element MUST contain only a 'prop' element.

 

remove

D:

Elements

Represents a property setting deletion, which has at least one prop child

Remove instructs that the properties specified in prop should be removed. Specifying the removal of a property that does not exist is not an error. All the XML elements in a 'prop' XML element inside of a 'remove' XML element MUST be empty, as only the names of properties to be removed are required.

 

prop

D:

Elements

All elements inside a 'prop' XML element MUST define properties related to the resource, This element MUST NOT contain text or mixed content.

 

DTD Notation


<!ELEMENT propertyupdate (set, remove)
<!ELEMENT set (prop*) >
<!ELEMENT remove (prop*) >
<!ELEMENT prop ANY>

リクエストサンプル


<D:propertyupdate xmlns:D="DAV:" 
    xmlns:dc="urn:x-dc1:xmlns" 
    xmlns:Z="http://www.w3.com/standards/z39.50/">
    <D:set>
        <D:prop>
            <Z:Author>Author1 update</Z:Author>
            <dc:hoge>fuga</dc:hoge>
        </D:prop>
    </D:set>
    <D:remove>
        <D:prop>
            <Z:Author/>
            <dc:hoge/>
        </D:prop>
    </D:remove>
</D:propertyupdate>
