Skip Query

概要

This query sets the number of records (say N, where N is a positive integer) to skip before it retrieves the remaining records (starting with Entry N+1) in a collection.

By using integer values ​​in $skip query, be positioned to start the acquisition of the data list is possible if the supplied value is greater than the number of data has been registered, you do not get the data.

If the number of data is larger than the upper limit of $skip, it is necessary to specify search criteria, such as $filter query for data acquired at $skip query is not sorted, the need to specify the sort conditions, such as $orderby query

※ If you specify a value larger than the upper limit or integer value other than $skip, it returns a "400 Bad Request".

 

The syntax of the $skip query is as follows:

$skip={number}

 

Item

Summary

Valid value

Remarks

{number}

Specifies the starting position of the returned feed.

(0: default) 0-100000 of numeric characters

 

 

For example:

To skip 10 cells from the cell list and show 11th cell onwards, the command is:

curl -X GET "https://fqdn/__ctl/Cell?$skip=10"\
 -H "Accept:application/json" -i -k -s
-H "Authorization:Bearer token"

Request Query

$skip={number}
