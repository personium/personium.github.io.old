NavProp経由登録API

概要

ユーザデータのEntityをNavigation Property経由で作成します。

必要な権限

write

制限事項

·           リクエストヘッダーのContent-Typeは全てapplication/jsonとして扱う

·           リクエストボディはjson形式のみ受け付ける

·           レスポンスヘッダーのContent-Typeはapplication/jsonのみをサポートし、レスポンスボディはjson形式とする

·           $formatクエリオプションにatom または xmlを指定した場合、エラーとはならないが、レスポンスボディのデータの保証はない

·           ユーザデータ制限事項

o     Edm.DateTime型のプロパティの有効範囲のチェックが適切に行われない

o     Edm.DateTime型の配列は未サポート

o     Edm.DateTime型のプロパティにSYSUTCDATETIME()を指定した場合、設定されるシステム時間が異なる場合がある

§      リクエストボディでの設定時とDefaultValueでの設定時（__published、__updatedは後者のタイミング）

o     1つのEntityTypeに対して作成出来るのは、DynamicProperty・DeclaredProperty・ComplexTypeProperty合わせて400個まで

o     文字列のみ設定可能
