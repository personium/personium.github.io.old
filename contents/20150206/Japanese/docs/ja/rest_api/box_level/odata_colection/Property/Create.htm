Property登録API

概要

ユーザーデータに指定するプロパティを定義する

必要な権限

write

制限事項

·           OData制限

o     リクエストヘッダーのContent-Typeは全てapplication/jsonとして扱う

o     リクエストボディはjson形式のみ受け付ける

o     レスポンスヘッダーのContent-Typeはapplication/jsonのみをサポートし、レスポンスボディはjson形式とする

o     $formatクエリオプションにatom または xmlを指定した場合、エラーとはならないが、レスポンスボディのデータの保証はない

·           個別の制限

o     関連対象のEntityTypeを使用したユーザーデータが存在する場合は、Nullableがtrueの場合のみ登録可能

o     Edm.DateTime型の配列は使用不可

o     Edm.DateTimeのDefaultValueの有効範囲のチェックが適切に行われない

o     1つのEntityTypeに対して作成出来るのは、DynamicProperty・DeclaredProperty・ComplexTypeProperty合わせて400個まで

o     isKey/UniqueKeyを設定しても、スキーマへの反映はされない

Valid values ​​for DefaultValue

DefaultValueの有効値はTypeの値（型定義）によって異なり、以下の定義となる型の異なる項目についても文字列で定義を行う

type値

桁数：0～51200 byte
「\」を使用する場合、「\\」で指定する必要がある

Edm.Int32

-2147483648 ～ 2147483647

Edm.Single

整数部分の桁数：1～5桁
小数部分の桁数：1～5桁

Edm.Boolean

true / false

Edm.DateTime

/Date(【long型の時刻】)/の形式で文字列で指定する
　【long型の時刻】の有効値は、-6847804800000(1753-01-01T00:00:00.000Z)～253402300799999(9999-12-31T23:59:59.999Z)
また、予約語として以下を指定可能
　SYSUTCDATETIME()：サーバ時間
