A

ACL

Access Control Listの略称。

オブジェクトに対する付与権限のリストとどのユーザがアクセスを許可されているかを定義する。

さらに、どのような制御命令の使用が許可されているか定義することも可能。（例：Read:読み込み許可、Write:記入許可、Read/Write:編集許可）

Account

特定のCellに属するユーザを意味する文字列。アカウント名やパスワードといった情報によって区別される。

Association

2つ以上のEntityType間の関係性を示す。（EntityTypeはRDBMSにおけるテーブルの概念とほぼ同義）

「1対1」「1対多」「多対多」の種類に分けて扱われる。

AssociationEnd

2つ以上のEntityType間の関係性（Association）を文字列で表現したもの。

Authentication

一般的なサーバ-クライアント認証モデルにおいて、クライアントは、サーバによる情報保持者の証明書を用いた認証を受けることによって、サーバ上にあるアクセス制限情報を要求する。

サードパーティのアプリケーションが制限情報にアクセスするには、サードパーティの証明書を情報保持者が共有をする。

B

Bar File

Box Archive Fileの略称。

Bar Install

Boxにおけるアーカイブファイルを作成すること。これにより、他の異なる場所にあるBoxのレプリカを導入することが可能。

Box

アプリケーションに用いるデータを格納する領域。一意の名前とスキーマURLを持つ。Cellは、Box未作成でも初期状態で1つのBox(default box)をもつ。

Box Level ACL

Boxに関するアクセス権限

C

Cell

「ペルソナコンテナ」の別名であり、データやアプリケーション共有の中心となるデータ領域。

Cell Control Objects

Cellとの関係をもつ機能の管理する（Role, Account, Box, ExtCell, ExtRole）

Cell Level ACL

Cellへのアクセスを要求される権限

Cell Manager

Cellの所有者のことで、そのCellへのアクセス権限や動作権限を持つ

Cell Profile

Cellを表現する名前。Cell Profileは、表示名、描写、イメージ、 スコープなどの関連情報をあらわす。

Collection

Cellにあるboxの中に格納されたのデータ集合の名称。「WebDAV」「Odata」「Engine Service Collection」の3種類がある。

Complex Type

副属性を伴った属性を持つこと。副属性の規則配列を含んでいる。

（例えば、住所はcomplex typeであり、副属性は、通り1・通り2国・郵便番号・都道府県・市区町村である）

Complex Type Property

Complex Typeの副属性のこと。例えば、 Complex Typeである住所においては、通り1・通り2国・郵便番号・都道府県・市区町村のことである。

CORS

Cross-Origin Resource Sharingの略称。

Webページにおいて、JavaScriptが他のドメインに対しXMLHttpRequestをすることを許可すること。

Cross Domain Access Control

手動または自動によるアクセスおよび複数の安全なドメインの間の転送を行う権限の付与したという情報の保証手段。

D

Data Management

ODataに関するサービスのひとつで、あるプロパティと関係したEntityTypeの新規作成、編集、消去の許可を行う。

E

Engine Service Collection

ユーザーがサーバー側のロジックをサービスコレクション内に新たに登録することが可能であり、そのロジックをEngine Service Collectionという。特別なコレクションを参照。

Entity

記録したデータの構造。例えば、名前・住所・性別といった情報のあつまりをあらわしたもの。

Entity Type

データの構造をEntity Data Model(EDM)であらわすためのデータ要素。Entity Typeは、上位概念の構造をあらわしたもの（顧客、注文内容など）

Environment

Cellの集合をあらわすデータ領域。

ETag

Entity Tagのこと。Webキャッシュの検証に用いる固有の識別子でクライアントの状況に応じたリクエストの送信が可能。

コンテンツの更新がない場合において完全なレスポンスを返す必要がないときにキャッシュの効果的な使用を高め、帯域幅を確保する。

Event

PCSの内部および外部から発生するインスタンス。

イベント受付APIにて定義された外部イベント、およびPCS内部で定義された内部イベントから構成される。

EventLog

外部および内部イベントの発生ログ。

Expand Query

1つのHTTP要求でエントリーのグラフを検索できるように、関連するエントリーをただ1つのURIで判別するクエリ。

External Cell

ある特定のセルの外にある、他のセル。External cellは、同じまたは異なる、あらゆるユニットに属すことができる。

External Role

セルからのユーザが外部のセルにアクセスするのを助けるセルの中に登録されたRoleのこと。

F

Filter Query

サポートしているODataクエリの1つ。EntitySet内でEntityの部分集合を特定するクエリ。

Format Query

サポートしているODataクエリの1つ。要求への応答がクエリオプションで指定したメディアタイプを使用しているものを特定するクエリ。

FQDN

Fully Qualified Domain Name（完全修飾ドメイン名）の略称。

インターネット上の特定のコンピュータやホストを一意に定義する完全なドメイン名をあらわす（例：host-name.domain-name.com）

Full-Text Search Query

全文検索クエリ。全文インデックスに基づいた自然言語検索を行う。

特定の言語（英語・日本語）のルールに則って、単語やフレーズでの動作する。

G

Grid View

各行がfieldを表し、各列がrecordを表すテーブルに値を表示する。

I

Implicit Flow

ウェブベースおよびデスクトップ両方のアプリケーション使用される標準のOAuth2.0フロー。このフローでは、クライアントは「request_type=token」の状態で、PCS portalへの認証要求をする。

以下を参照のこと

http://msdn.microsoft.com/en-us/library/live/hh243647.aspx#implicitgrant

Inlinecount Query

サポートしているODataクエリの1つ。コレクションにおける、エンティティの数のカウントを表示するクエリ。

L

List View

スキーマ管理オプションの1つ。異なったEntity Typeの間に作られたassociationを見ることができる。

Link Registration

ユーザデータ内において、指定したEntity Typeを持つEntityと、同じもしくは異なるEntityTypeを持つ他のEntityとを関連づけること。言い換えると、2つのEntity間の関係性(Link)を発行すること。

M

Main Box

Cell作成時にデフォルトで作成される、”__”(アンダーバー2つ)と名づけられたBox。動作は他のBoxと同様となる。

アプリケーションデータを保管する目的以外にも、そのCellの固有情報（json形式を用いる）を格納する。

Message

Cell間の関係性を発行するメッセージの送受信を行う仕組み。

MKCOL

リクエストURIによって指定された位置に新たなコレクションを生成するメソッド。

Multiplicity

多重度。Entity Type作成時に、関係するEntityTypeの数を指定する。

Association endの多重度の表記は、2つのEntity Typeの間の関係では、「1」、「0..1」(0または1)、「*」(多数)のいずれかを取る。

O

OData

Open Data Protocolの略称。標準データアクセスプロトコルであり、データ資源に対し標準的なCRUDアクセスを提供する。

Orderby Query

サポートしているODataクエリの1つ。ユーザーが特定した順序でソートされた値を表示する。デフォルトでは昇順。

P

Password authentication

パスワード認証。正規のユーザーがPCS Portalにログオンする手段。ユーザー名はデータデベースにおいてパスワードと照合される。

ユーザーに入力されたパスワードが名前と一致するときユーザーはシステムとのアクセスが可能になる。

PCS

ペルソナコンテナサーバ（Persona Container Server）の略称。PCSは、情報連携のための個人データ領域であり、多デバイス間のデータ同期の中心である。

Property

各EntityTypeの列頭の値。例えば「顧客」というEntityTypeには「ID」「名前」「住所」といったPropertyが考えられる。

Privileges

Cell内に定義されたRoleに対し、特定のRoleに紐付けられたBox内部のデータにアクセスする権利・権限・特権。

PCSではACL (access control level)の設定によって定義される。

R

Raw View

JSON形式で書かれたEntityをそのままの形式で表示すること。

RBAC

Role-based access controlの略称。

異なるユーザー（アカウント）に属するRoleに対し、制限つきアクセスを実現する認証機構であり、あらゆるCellに設定される。

Received Message

特定のCellからの、リンク発行リクエストの受信または拒否のためのメッセージを受け取ること。

Refresh Token Authentication

新たなアクセストークンを再発行するプロセス。発行されたトークンの持続時間は24時間である。

Relation

Cellとその外部のCellとの関係。同じまたは異なるユニットに属している外部のCellの間で関係を確立できる。

常にCell固有の関係性になるようにBoxに関連づけられる。

Relation Class URL

アプリケーションと定義される関係リソースへのURL。RelationクラスURL構造は以下の通り:

${Schema URL}/__relation/__/${RelationName}

Relation Instance URL

要求が1つ以上のExternal Cell(s)に送られる特定のRelationのURL。

Relation Instance URLの構造は以下の通り:

${Cell URL}/__relation/${BoxName}/${RelationName}

RequireSchemaAuthz

ユーザがschemaにアクセスできる権限を持つか否かに関係なく、schemaレベル認証を有効にする。

Resource Type

Collectionの型。

Role

すべてのCellに対して定義される「役割」をあらわす変数。（例：administorator、teacher、student）

どのアカウント(ユーザ)が、そのCellにアクセスすることができるかを指定できるようになるので、

Cellのアカウント所有権を規定するアクセス権限の異なった設定を持つことができる。 常に固有になるようにBoxに関連づけられる。

Role Class URL

Trance cellトークン内部に蓄積させるためのRoleのURL。

Role ClassURLの構造は以下の通り:

${schema URL}/__role/__/${RoleName}

Role Instance URL

特定のCellに登録されたRoleの、現行状態をに提供するURL。Role Resource URLと同様。Schemaは以下の通り:

${Cell URL}/__role/${BoxName}/${RoleName}

Role Mapping

コントロールオブジェクト（Account、ExtCell、ExtRole、Relationなど）でRoleを定義する。

Cell間の参照したデータ内にアクセスする。

S

Schema Management

List view、Association viewの2種類のタイプの視点がある。

(1)List view：Entity Typeのプロパティを作成する。

(2)Association view： 2つ以上のEntityタイプを関連づけるか、またはリンクし、それらのデータへのアクセスを補助する。

Schema URL

存在するすべてのSchemaのURL。

Select Query

サポートしているODataクエリの1つ。あらゆるデータテーブルに対し、特定のPropertyのセットをリクエストすることが可能。

例： $select=Name, Address

Sent Message

目的のCellに対し、自身のCellから、Relationの定義を発行するためにメッセージを送信すること。

Service Collection

ユーザー定義のサーバーサイドロジックを実行するサービスのコレクション。

Service Registration

存在するコレクションに、ユーザー定義のサーバーサイドロジックを登録すること。

Skip Query

サポートしているODataクエリの1つ。コレクション内に残った記録データを検索する前に、記録データの個数分スキップするクエリ。

Social Graph

絵の形式で複数のCell間の関係について表現したもの。 PCS Portalのソーシャルグラフは2種類（unforcussed:全体表示、forcussed:Cell視点表示）で描画される。

T

Token

認証プロセスの一つ。PCS Portalのデータやリソースにアクセスするためには、この方式でユーザが認証される。

これらのトークンは、それぞれのデータとリソースに対して、トランザクションを認可するというすべてのその後のAPIリクエストに含まれている。

トークンがない場合や、トークンの期限が切れている場合には、PCS Portalはリクエストを拒否する。

Token Authentication

データにアクセスするのに必要となるAccessトークンによる認証方法。以下の2種類:がある。

(1)Cellのローカルのトークン：単に認証されたセルに関するリソースにアクセスする際に使用

(2)Trans cellトークン：他のセルと同様に認証されたセルに関するリソースにアクセスする際に使用

セキュリティ上の観点から、Accessトークンは1時間で変更される。

Top Query

サポートしているODataクエリの1つ。最大記録数を指定しその個数分を返す。抽出データは、セット内の最初のN個分（Nは整数値）となる。

例：最初の5個、昇順

U

Unit

PCS基盤サーバ内で、1つ以上のEnviromentから構成されるデータ領域。完全修飾ドメイン名（FQDN）を持ち、それは絶対ドメイン名として参照される。

Unit Control Objects

Unitの管理者として、Cellの作成や管理をするためのオブジェクト。

Unit User

ある特定のユニットの所有者、管理者。ユニット内部のCRUD（Create/Read/Update/Delete）の権限を持つ。

User Authentication

それぞれのCellのユーザを有効にするプロセス。Account 認証 と Transcell認証がある。

(1)Account 認証 ：ユーザとそのアカウントがそれぞれのCellに認証されるか否かに関係なく、有効にされる

(2)Trans cell認証：ユーザが他のCellと同様にそれ自身のCellで有効にされる

W

WebDAV

Web-based Distributed Authoring and Versioningの略称。

それはHTTPの拡張であり、Webサーバ内のドキュメントやファイルを、異なるユーザで共同で執筆するためのプロトコル。

PCS Portalは、WebDAVコレクションはファイルやフォルダであり、またCRUDの機能が動作する。
