ロール管理

ロールは各セルで定義され、それによってユーザーはセルにアクセスができます。

ロールはセルのアカウント所有に応じたアクセス権を持つことができます。

このアクセス制御は、RBAC (Role Based Access Control)の概念に基づいており、権限は、特定のセルへアクセスする各アカウントに割り当てられます。

  • ロールの種類

ロールには以下の種類があります。

·         Boxに紐づくロール

·         Boxに紐づかないロール

ロールを定義することで、指定されたアカウント(ユーザ)が特定のセルにアクセスすることができます。

·        データアクセスを含むセルへのあらゆるアクセスは、付与されたRoleに応じて制御されます。

·        セルにアクセスするためには、指定されたアカウント(ユーザ)とRoleを定義しなければなりません。

·        Account/ExtCell/Relation/ExtRoleに対しRoleを紐付けることによって、利用者主体(User entity)にRoleを付与することが出来ます。

·        Roleを定義するためには、RoleをBoxに関連づける($links登録)必要があります。(関連付けない場合、Roleは自動的にMain Boxに関連付けられます。)

·        セルにアクセスするには、利用者にRoleを定義する必要があります。

·        セルのアカウントまたはExtCellのユーザがセルにアクセスするRoleを定義できます。

ロールに関して以下の操作が可能です:

    1. 作成 ( Role登録 API を参照)

    2. 表示 ( Role一覧取得 APIを参照)

    3. 更新( Role更新 APIを参照)

    4. 削除 (  Role削除 APIを参照)

    5. Account, External Cell, Relation, External Roleとの$links作成
