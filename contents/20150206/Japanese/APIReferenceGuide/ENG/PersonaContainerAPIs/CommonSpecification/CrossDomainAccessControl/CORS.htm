CORS Support

  • Overview

API of all of persona container, meant to be used in principle CORS (Cross Origin Resource Sharing) with some exceptions.

If you do not have a description as a special exception, I will return the response header following all API endpoint on request specifically.

  1. Access-Control-Allow-Origin

  2. Access-Control-Allow-Methods

  3. Access-Control-Allow-Headers

Thus, for performing the HTTP asynchronous communication with the server script language Web browser installed, such as JavaScript, you can use the XMLHttpRequest Level 2.

  • Return of Access-Control-Allow-Origin response header

In the API of all, I will return * as Access-Control-Allow-Origin response header.

Access-Control-Allow-Origin response header is to return the domain that allow access, but persona container such behavior in order to allow access from all domains.

The following is an example of a response header Access-Control-Allow-Origin.

 

Access-Control-Allow-Origin:*

  • Response to the pre-flight requests from the browser

And the behavior of the following as a response to pre-flight requests from the browser, when you issue the request of the OPTIONS method in the API of all.

  1. Return of Access-Control-Allow-Methods response header

  2. Return of Access-Control-Allow-Headers response header

  • Return of Access-Control-Allow-Methods response header

When you issue the request of the OPTIONS method in the API of all, it returns the Access-Control-Allow-Methods response header.

Contents of the Access-Control-Allow-Methods Responsive header will change as follows depending on the contents of the Authorization header and access control settings of the API.

The resources (read possible without Authorization header published)

I will return all the HTTP methods that the resource supports.

Private (non-public) OPTIONS request without the Authorization header to the resource

I will return all the HTTP methods that the PCS can support.

I will return all the HTTP methods that the resource supports.

 

The request Authorization header with, it can sometimes be an error of the following, depending on the contents of the header.

  1. 401: the contents of the Authorization header is invalid.

  2. Request is not accepted by the situation of access control, such as: 403. (. Reading is not allowed)

I will return the request method that is allowed inside the request URL.

The following is an example of a response header Access-Control-Allow-Methods.

Access-Control-Allow-Methods:GET,OPTIONS

  • Return of Access-Control-Allow-Headers response header

In the API of all, if you specify the Access-Control-Request-Headers in the request header, it returns the Access-Control-Allow-Headers.

So that you can send all of the header that the client wants, what is returned in the Access-Control-Allow-Headers response header, and returns as the value that is specified in the Access-Control-Request-Headers.

The following is an example of the response header if you specify a "sample" in the Access-Control-Request-Headers.

Access-Control-Allow-Headers:sample
