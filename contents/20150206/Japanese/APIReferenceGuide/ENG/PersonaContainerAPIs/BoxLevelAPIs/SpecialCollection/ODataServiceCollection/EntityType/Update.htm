Update EntityType

  • 概要

This API updates the existing information of EntityType

  • 必要な権限

write

  • 制限事項

1        'Content-Type' header of the request is treated as application/json

2        Request body is accepted only in json format

3        'Content-Type' header of the response only supports application/json. The response body appears in json format

4        If you specify the XML or atom as a $format query option, it is not an error, there can be no assurance of the data in the response body

  • CURL Command

Syntax

curl -X PUT "https://fqdn/CellName/BoxName/OdataName/$metadata/EntityType('EntityTypeName')"\
 -d "{\"Name\":\"NewEntityTypeName\"}"\
 -H "If-Match: *"\
 -H "Authorization:Bearer token" -i -k -s

CURLサンプル

curl -X PUT "https://fj.baas.jp.fujitsu.com/CompanyA/Finance/Salary/$metadata/EntityType('Perks')"\
 -d "{\"Name\":\"Bonus\"}"\
 -H "If-Match: *"\
 -H "Authorization:Bearer PEFzc2VydGlvbiB4bWxucz0idXJuOm9hc2lzOm5hbWVzOnRjOlNBTUw6Mi4wOmFzc2V" -i -k -s

 

Where:

fqdn = domain name of the specific Cell, such as "fj.baas.jp.fujitsu.com"

CellName = Cell name, such as CompanyA

BoxName = Box name, such as Finance

OdataName = OData name, such as Salary

EntityTypeName = Entity type name, such as Perks

NewEntityTypeName = Entity type name, such as Bonus

token = Token value, such as "PEFzc2VydGlvbiB4bWxucz0idXJuOm9hc2lzOm5hbWVzOnRjOlNBTUw6Mi4wOmFzc2V"

Note:

Please note that this is a dummy token value. It automatically gets generated at the time of authentication.
