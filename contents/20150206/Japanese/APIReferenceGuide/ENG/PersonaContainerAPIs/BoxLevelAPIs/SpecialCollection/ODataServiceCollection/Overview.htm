DynamicProperty

If for EntityType, the registered user data without dynamic schema definition, key register is treated as DynamicProperty (Property dynamic).

·         DynamicProperty operates in the following specifications.

o        Type of dynamic property does not appear in the schema information ($ metadata).

o        Because of the above, it does not come can see the user data OData browsing tool is common in (ODataExplorer etc.).

o        Can not handle user data array, DateTime ·ComplexType virtual array etc.

o       It is also possible to co-exist in the EntityType of one the Property a schema-less Property and that are schema set, but for the Property that has been created in the schema-less, the above limitation applies.

o       If when you register the user data, make a mistake the Property name that is set schema, because a new DynamicProperty from being created in the name, be careful.

AssociationEnd

AssociationEnd is OData object to define the end point of the relationship between the EntityType (Association).

The basic steps that define the nature of the relationship between EntityType is as follows.

  1. And against one of the EntityType that defines the relationship, to register the AssociationEnd. At this time, I set at the same time multiplicity of the EntityType one also (Multiplicity) anymore.

  2. To the other end of the EntityType that defines the relationship, I register the AssociationEnd. At this time, I set at the same time multiplicity of the EntityType one also (Multiplicity) anymore.

  3. For AssociationEnd you have registered, to register the link in the $ links.

·         AssociationEnd works with the following specifications.

  • You can not AssociationEnd in one definition, to define the Association of two or more.

    • Mounted on the string in $ linksAPI, it is a duplicate error.

Multiplicity (Multiplicity)

Multiplicity (multiplicity) in the context of the EntityType between, how much from the user data of the EntityType of the relationship partner
is the attribute value of AssociationEnd to define whether you can put a relationship.

Multiplicity can be set are as follows.

Description

Meaning

One

User data exactly one corresponding to the relationship target

0-1

Or 0 for target relationship, user data one corresponding

*

Or 0 for target relationship, user data of a plurality of corresponding
