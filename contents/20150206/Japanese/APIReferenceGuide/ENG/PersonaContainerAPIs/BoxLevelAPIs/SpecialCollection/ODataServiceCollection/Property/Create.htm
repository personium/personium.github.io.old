Register Property

  • 概要

This API defines the properties that you specify for the user data

  • 必要な権限

write

  • 制限事項

·         リクエストヘッダーのContent-Typeは全てapplication/jsonとして扱う

·         リクエストボディはjson形式のみ受け付ける

·         レスポンスヘッダーのContent-Typeはapplication/jsonのみをサポートし、レスポンスボディはjson形式とする

·         $formatクエリオプションにatom または xmlを指定した場合、エラーとはならないが、レスポンスボディのデータの保証はない

·         If the user data using the EntityType associated target exists, Nullable can be registered only if the true

·         If you want to create a Property and Property of the same name that you have deleted, can be registered only if the same type
(you will need to delete the EntityType, to re-create the Property If you want to change the type)

·         Edm.DateTime array of type can not be used

·         Check the scope of the DefaultValue of Edm.DateTime is not done properly

·         You can create one for EntityType, Property · ComplexTypeProperty together until 400

·         If you set the isKey / UniqueKey, reflected in schema is not
