Glossary

A

ACL

[General] Stands for Access Control List.

It specifies a list of permissions associated to an object and which users are granted access to the object. It also specifies what operations are allowed on given objects, such as Read, Write, and Read/Write.

Account

[personium.io] Refers to users belonging to the particular cell. It stores information, such as account name and password. See Cell Control Objects.

Association

[OData] Refers to relationships between two or more EntityTypes. EntityType is similar to tables in the RDBMS concept.

There are several types of associations, such as One to One, One to Many, and Many to Many. See OData Service Collection.

AssociationEnd

[OData] Refers to association names that represents the relationship between two or more EntityTypes. See OData Service Collection.

Authentication

[General] In the traditional client-server authentication model, the client requests an access-restricted resource on the server by authenticating with the server using the resource owner's credentials. In order to provide third-party applications access to restricted resources, the resource owner shares its credentials with the third party.

B

Bar File

[personium.io] Stands for Box Archive File.

It is an archive file that summarizes the contents of the configuration Box.

By representing the hierarchical structure of the WebDAV collection in a hierarchical structure in the folder, to store the file in the hierarchy each, to hold the file to be stored in the WebDAV. See Bar File.

Box

Refers to the unit that keeps data of an application. Box has a unique name and schema URL. If no box is created, cell will have at least one default box. See Box Installation.

Box Install

[personium.io] Refers to a bar file that is used to create a Box. See Box Installation.

Box Level ACL

[personium.io] Refers to permission required to access the Box. See Access Control Method.

C

Cell

[personium.io] Refers to an alias of personium.io, which is treated as a center for data and application(s).

Cell Control Objects

[personium.io] Manages functions associated with the Cell, such as Role, Account, Box, ExtCell, and ExtRole. See Cell Control Objects.

Cell Level ACL

[personium.io] Refers to permission required to access the Cell. See Authentication Model.

Cell Manager

[personium.io] Refers to the owner of the specified Cell. The Cell Manager has some set of rights to access and perform function on the Cell.

Cell Profile

[Personium.io Portal] Refers to the representation of a Cell. The Cell Profile shows cell related information, such as display name, description, image, and scope.

Collection

[WebDAV] Represents to a group of data, existing inside a box of the specified Cell. There are three types of Collections: WebDAV, Odata, and Engine Service Collection.

Complex Type

[OData] Refers to an attribute that can be further broken into sub-attributes. The Complex Type contains an ordered collection of sub-attributes. For example the address is complex type, which contains sub-attributes, like street 1, street 2, city, state, zip, and country.

Complex Type Property

[OData] Refers to the sub-attributes in the Complex Type. For example, a Complex Type address can have Complex Type Property, like street 1, street 2, city, state, zip, and country.

CORS

[General] Stands for Cross-Origin Resource Sharing.

CORS allows JavaScript on a Web page to make XMLHttpRequests to another domain. See CORS.

For more information, visit http://www.w3.org/TR/cors/.

Cross Domain Access Control

[General] Refers to the means of information assurance that provides the ability to manually or automatically access or transfer between two or more security domains.

See Cross Domain Policy file.

D

E

Engine Service Collection

[WebDAV] Refers to a special collection, which allows user to register a server side logic to an uploaded file under service collection.

Entity

[OData] Refers to a record structure.

Entity represents a collection of information, such as a name, address, and gender in the EntityType.

Entity Type

[OData] Refers to the building block for describing the structure of data with the Entity Data Model (EDM).

An entity type represents the structure of top-level concepts, such as customers or orders.

Environment

[Personium.io Portal] Refers to a group of Cells.

ETag

[personium.io] Stands for Entity Tag.

ETag is a unique identifier that is used for Web cache validation, which allows a client to make conditional requests. This allows caches to be more efficient, and saves bandwidth, as a web server does not need to send a full response if the content has not changed.

Event

[personium.io] Refers to an instance that can occur internally as well as externally in the personium.io portal. The event consists of External Event and Internal Event. The External Event represents the events that are accepted. The Internal Event refers to the process of executing the APIs within personium.io.

EventLog

[personium.io] Collaborates the content of all the events occurred in the event reception desk

Expand Query

[OData] Identifies related Entries with a single URI such that a graph of Entries could be retrieved with a single HTTP request.

External Cell

[personium.io] Refers to other cells outside a particular cell. An external cell can belong to the same unit or to the another unit.

External Role

[personium.io] Refers to the registered role within a cell that helps users from a cell to access the external cell.

F

Filter Query

[OData] It is a supported OData query, which identifies a subset of the entities in the EntitySet.

Format Query

[OData] It is a supported OData query, which specifies that a response to the request should use the media type specified by the query option.

FQDN

[General] Stands for Fully Qualified Domain Name. It is the complete domain name for a specific computer or host existing on the Internet.

Full-Text Search Query

[OData] It is a supported OData query, which performs linguistic searches against text data in full-text indexes. It operates on words and phrases based on rules of a particular language such as English or Japanese.

G

I

Implicit Flow

[OAuth2.0] It is a standard OAuth 2.0 flow used by both Web-based and desktop applications. In this flow, the client makes an authorization request to the personium.io portal with request_type=token. For more information, see http://msdn.microsoft.com/en-us/library/live/hh243647.aspx#implicitgrant.

Inlinecount Query

[OData] It is a supported OData query, which displays the count of the number of entities in the collection.

L

M

Main Box

[personium.io] Works similar to other box(es) in the cell. It is denoted by name “__” (underscore underscore) and is created by default when a cell is created. Other than serving purpose to store application data, the main box also serves to store profile information of the cell (in json format)

Message

[personium.io] Refers to the mechanism of sending and receiving messages between cells to establish relations among themselves. See Message Management.

Multiplicity

[OData] Indicates the nature of association between the Entity Types while creating it. An association end multiplicity can have a value of one (1), zero or one (0..1), or many (*) indicating relationship(s) between two entity types.

O

OData

[personium.io] Stands for Open Data Protocol.

It is a standard data access protocol that provides standard CRUD access to a data source. Visit http://www.odata.org/.

OData Service Collection

[WebDAV] Refers to a special collection for working with OData. See OData user model.

Orderby Query

[General] It is a supported OData query, which allows users to view values that is sorted in specific order. Ascending order is the default order. See Orberby Query.

P

Property

[OData] Refers to the column headings of the respective Entity types. For example, a Customer entity type may have properties such as CustomerId, Name, and Address.

Privileges

[General] Stands for permission, right, or benefit granted to roles defined in a cell, to access data inside the box associated with specific role(s). Privileges are defined in personium.io portal as ACL (access control level) settings. See Authorization Model.

R

RBAC

[General] Refers to Role-based access control (RBAC) attached to each Cell, is the Authentication mechanisms of the Cell to provide customized access to different roles belonging to different users (accounts). See Authorization Model.

Received Message

[personium.io] Refers to receiving a message from a particular Cell for accepting or rejecting a request for establishing a relation.

Refresh Token

[personium.io] Refers to the process of re-issuing an access token. See Authentication Model.

Refresh Token Authentication

[personium.io] Refers to the process of reissuing the new Access token. The duration of the Refresh token is 24 hours.

Relation

[personium.io] Refers to the relationship between cell and its external cells. Relation can be established between external cells belonging to the same unit or any other unit. Relation is always associated with a box to make it unique. See Cell Control Objects.

Relation Class URL

[personium.io] Points to the relation resource that is defined as application. The Relation class URL structure is as follows:

${Schema URL}/__relation/__/${RelationName}

Relation Instance URL

[personium.io] Represents the URL of the particular Relation for which a request is sent to one or more External Cell(s). The structure of the Relation instance URL is:

${Cell URL}/__relation/${BoxName}/${RelationName}

RequireSchemaAuthz

[WebDAV] Validates schema level authentication whether user is allowed to access schema or not.

Resource Type

[WebDAV] Refers to type of collection.

Role

[personium.io] Role(s) are defined for each cell, such as administrator role, teacher role, and student role, through which account (user) can get access to a cell. A role can have different set of access privileges defining an account ownership of the cell. A Role is always associated with a box to make it unique. See Cell Control Objects.

Role Class URL

[personium.io] Refers to the Role URL which should be internally accumulated in the Trans cell token. The structure of the Role class URL is:

${schema URL}/__role/__/${RoleName}

Role Instance URL

[personium.io] Provides the actual state of the registered Role in a particular cell. The Role instance URL is the same as Role resource URL. The schema is as follows:

${Cell URL}/__role/${BoxName}/${RoleName}

ROPC

[OAuth2.0] Stands for Resource Owner Password Credentials. One of the approval process defined in OAuth2.0.

For more information, visit http://openid-foundation-japan.github.io/draft-ietf-oauth-v2-draft22.ja.html#anchor7.

S

Schema URL

[personium.io] Refers URI of any existing schema

Select Query

[OData] It is a supported OData query, which allows you to request a limited set of properties for each data table. For example, $select=Name, Address.

See Select Query.

SentMessage

[personium.io] Allows you to request the destination cell(s) to establish a defined relation with your cell, through sending a message. See Message Model.

Service Collection

[WebDAV] Refers to service collection executing the server-side user-defined logic.

Service Registration

[WebDAV] Refers to register any server side uesr-defined logic with existing collection

Skip Query

[OData] It is a supported OData query, which sets the number of records (say N, where N is a positive integer) to skip before it retrieves the remaining records (starting with Entry N+1) in a collection. See Skip Query.

T

Token

[General] It is a part of Authentication process where a user is authenticated to access data or resource in personium.io Portal. These tokens are included in all subsequent API requests to authorize transactions against the respective data and resources. Personium.io Portal will deny the request if you do not include the correct authorization token or it the token has expired.

See Authentication Model.

Token Authentication

[General] Refers to an Access token that is required to access data. There are two types of token authentication: Cell Local Token and Transcell Token. The Cell Local token is only valid to access the resource of the authenticated cell. The Transcell token helps in accessing resources of the authenticated cell as well as other cells.

For security concerns, the Access Token is short-lived and it gets expired in one hour. See Authentication Model.

Top Query

[OData] It is a supported OData query, which determines the maximum number of records to return. A data subset is formed by selecting only the first N items of the set, N being a positive integer. This can be combined with other query, such as OrderBy. For example, first five objects in ascending order. See Top Query.

U

Unit

[personium.io] A unit in an infrastructure that uses personium.io as a base to create one or more environments. Each unit has its own Fully Qualified Domain Name (FQDN), sometimes also referred as an absolute domain name.

Unit Control Objects

[personium.io] Allows you to create and manage one or more cells.

Unit User

[personium.io] Refers to the owner of the particular Unit. The Unit user has a right to perform CRUD operations, which includes create, read, update, and delete cell(s).

W

WebDAV

[General] Stands for Web-based Distributed Authoring and Versioning.

It is an extension of HTTP and a protocol for collaborative authoring between users in managing documents and files that are stored on Web servers. In personium.io portal, WebDAV collections refer to files and folders and CRUD operations performed on it.

For more information, visit http://www.webdav.org/.
