Box Installation

Overview

This API allows you to install the Box at the specified path using the bar file. For the bar file format, see bar_File (bar file). 

This API adopts the asynchronous transmission method, which means that after accepting the Box installation, it returns immediately. Therefore, to verify the box installation status, use the following API.

·           (Box metadata acquisition) Box Metadata

If the Box is not installed successfully, you can confirm the box installation situation in this API and can view the reason that caused this error.

The following code snippet shows the Box installation acceptance and status check:

 

Example of the Box Installation call (For 30 seconds of polling at client side)  

1 Box installation acceptance.

    - MKCOL / {Cell} / {box}

2 Box installation status check.

    - GET / {Cell} / {box} -> return in the "process".

    - 30 seconds polling

3 GET /{cell}/{box} -> Returns "processing complete" .

※ Polling should continue till the No. 2 process is complete.

Required Privileges

box-install

Limitations

Box installation target box limit

·           If there is a Box with the same name, you cannot install it again.

·           If the Box with the same scheme URL exists, you cannot install the bar files.

·           You cannot install box to the Main box.

·           You cannot specify a null field in the schema of "/ bar/00_meta/00_manifest.json": bar file.

Bar files limitations

·           File size of the box installable can be equal to or less than the upper limit.

You cannot install the box if it exceeds the upper limit.

§      100MB: size of bar files

§      10MB: uncompressed file size of bar file entry

·           In Bar_File each entry is stored in the order in which they are defined.

Box installation log detail limitation

Log details of the Box installation is captured in the EventBus of the Cell, attached with the Box of target Box install.※1

Therefore while referring the log details, Get log file API should be used.

·           In order to use the Get log file API, "log-read" permission is required.

·           For mixing the event log out of Box install, the value of "RequestKey" field need to be filtered.  

Filter after retrieving the value of "Requestkey" field of "MKCOL" that could be retrieved by "action" field.

Other Limitations

·           During the installation process, data manipulation cannot be performed to resources which are included under the bar files to be installed

·           It will not be rolled back when an error occurs in the box during installation.
