Received Message Approval API

Overview

This API approves the message

Type of message if the message is to approve

Changes the read/unread status of messages

Type of the message is approved if the req.relation.build

RequestRelation of the message to be approved, according to the value of RequestRelationTarget, to register the relationship, you want to change to approve/reject the state of the message

Type of the message is approved if the req.relation.break

RequestRelation of the message to be approved, according to the value of RequestRelationTarget, remove the relationship, you want to change to approve / reject the state of the message

Required Privileges

message

requires only relationship registration/deletion approval: social

Limitations

·           'Content-Type' header of the request is treated as application/json

·           Request body accepts only json format

·           'Content-Type' header of the response only supports application/json. The response body appears in json format

·           If you specify the XML or atom as a $format query option, it is not an error, but there can be no assurance of the data availability/correctness in the response body
