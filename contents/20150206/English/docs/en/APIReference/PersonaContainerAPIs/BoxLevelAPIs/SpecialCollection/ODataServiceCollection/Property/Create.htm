Register Property

Overview

This API defines or creates the properties that you specify for the user data

Required Privileges

write

Limitations

Common Limitations

None

OData Limitations

·           'Content-Type' header of the request is treated as application/json

·           Request body is accepted only in json format

·           'Content-Type' header of the response only supports application/json. The response body appears in json format

·           If you specify an atom as a $format query option, it is not an error, there can be no assurance or correctness of the data in the response body

Individual Limitations

·           When the user data, using the target relation of EntityType exists, registration will be possible only if the Nullable is true.

·           You cannot use Array of Edm.DateTime type

·           For one Entity Type, up to 400 will be possible to create for all together DynamicProperty・DeclaredProperty・ComplexTypeProperty

·           Schema won't be reflected even after setting the isKey/UniqueKey.
