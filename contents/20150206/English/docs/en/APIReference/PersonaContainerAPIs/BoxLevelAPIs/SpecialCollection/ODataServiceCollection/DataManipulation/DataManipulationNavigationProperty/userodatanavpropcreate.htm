Register using NavProp

Overview

This API creates an Entity of user data through Navigation Property

Required Privileges

write

Limitations

Common Limitations

None

OData Limitations

·           'Content-Type' header of the request is treated as application/json

·           Request body is accepted only in json format

·           'Content-Type' header of the response only supports application/json. The response body appears in json format

·           If you specify an atom as a $format query option, it is not an error, there can be no assurance or correctness of the data in the response body

User Data Limitations

·           Array of Edm.DateTime type not supported

·           When specifying the SYSUTCDATETIME () in Edm.DateTime type properties, there is a possibility that the system time set is different

o     During the initialization time of request body and DefaultValue, (__published, __updated are the latter case timing)

·           It is possible to create up to 400 DynamicProperty・DeclaredProperty・ComplexTypeProperty for each EntityType

·           When setting the Property of ComplexType array type, it is not possible to specify a null array elements

Treatment of a number

Fractional value (Edm.Single type)

Handling of the case to get the UserOData in JSON format are as follows:

·           The value of the fractional part of 5.0, etc. is 0, It will return as an integer value

Numeric (Edm.Double type)

※ treatment of type Double of the personium.io follows the specification of the Java Double

·           Handling of the case to get the UserOData in JSON format are as follows:

o     The value of the fractional part of 5.0, etc. is 0, I will return as an integer value

o     Numerical value of four decimal places, and returned in three-point after rounding

·           Information about what is returned

o     If the number of input values ​​at the time of registration is accurate to double or more, performs data registration and rounded to the precision to be managed as floats internally, but the range that does not occur informative drop on output In and outputs the converted fixed-point number representation

o     When using the input fixed-point number output, the identity of the original number and the number of inputs is guaranteed
