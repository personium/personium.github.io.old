Configure Access Control

Overview

This API provides access control information for the selected entity

Required Privileges

write-acl

Limitations

Common Limitations

None

WebDAV Limitations

·           Maximum depth of the hierarchy of the collection 50

·           The maximum value of the child number of elements in each collection under 10,000

·           For count of hierarchy

Second level ... 10000 hierarchy eyes first layer

box/WebDavCollection1/WebDavCollection2 / ... / WebDavCollection10000/WebDavfile

 

The first layer and the collection of Box directly below, to count the second layer, and three-tier first ... below.

The file that you create in the underlying collection, it does not count as a hierarchy.

Functional Restriction

When you make an ACL configuration, it will be updated in the form that will be overwritten the existing ACL settings.

·           Function to cancel the ACL settings (deny)

·           List acquisition of privilege that can be configured in the ACL

Request Sample


<?xml version="1.0" encoding="utf-8" ?>
<D:acl xmlns:D="DAV:" xmlns:dc="urn:x-dc1:xmlns" 
       xml:base="https://example.com/testcell1/__role/box1/" 
       dc:requireSchemaAuthz="public">
    <D:ace>
        <D:principal>
            <D:all/>
        </D:principal>
        <D:grant>
            <D:privilege><D:read/></D:privilege>
        </D:grant>
    </D:ace>    
    <D:ace>
        <D:principal>
            <D:href>role</D:href>
        </D:principal>
        <D:grant>
            <D:privilege><D:read/></D:privilege>
            <D:privilege><D:write/></D:privilege>
        </D:grant>
    </D:ace>
</D:acl>
