$filter Query

Overview

Specify the $filter to put a condition for retrieving the list

Operator Support

※ When you specify a URL-encoded query requires the effective value of the property being searched ※ The effective (underscore characters) alphanumeric characters and _.
※ If you want to include a single quote in the search string, you can specify the search string in single quotes to write "''" two single quotes can be
if you specify the name of the property that does not exist in the ※ $ filter, do a search, including the search criteria
※ __updated, if you specify the __ published, specify (numbers in parentheses "/ Date ()") in UNIX time

Operator

Description

Example

Remarks

eq

Equal sign

$filter=itemKey eq 'searchValue': the case of a string
$filter=itemKey eq 10: the case of a number

The corresponding string, number, boolean, to NULL

gt

Greater than

$filter=itemKey gt 1000

Corresponding string, a number

ge

Or more

$filter=itemKey ge 1000

Corresponding string, a number

lt

Smaller than

$filter=itemKey lt 1000

Corresponding string, a number

le

Below

$filter=itemKey le 1000

Corresponding string, a number

and

Logical AND

$filter=itemKey1 eq 'searchValue1' and itemKey2 eq 'searchValue2'

 

or

Logical sum

$filter=itemKey1 eq 'searchValue1' or itemKey2 eq 'searchValue2'

 

()

Priority group

$filter=itemKey eq 'searchValue' or (itemKey gt 500 and itemKey lt 1500)

If there is only one, parentheses are ignored

Search specification of each property type

Edm.String type

Search by converting to a string type a keyword that you specified in the search

Edm.Int32 type

Null or integer values ​​can be specified to search converted to a numeric type if it is an integer value of type string

Edm.Single type

Specified integer, or null decimal value can do a search by converting to a small value if the specified integer, the fractional value of type string

Edm.Boolean type

Can be specified only null or boolean

Notices

※ URL must be encoded when specifying a query

※ effective value of the searched property are: alphanumeric characters, - (hyphen), _ (single-byte underscore).

※ Single cote 「'」could be specified in the search, by inputing single cote 「'」 twice 「' '」.

※ If specified property name does not exist in the $ filter, do search including the search criteria

※ Specify by UNIX time [the numbers of parentheses in "/ Date ()"], when specifying __updated, __published.

※ The maximum value for the searchable data by exact search, range search, forward matching are as follows:

     When searching, if the searched data exceeds the maximum value of the property, the value will not be included in the search results .
