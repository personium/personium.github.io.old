$top Query

Overview

By using integer values in $ top query, can limit the retrieve data list number.

If the number of registered data are less than the retrieved data then, it will retrieve all the data

If the number retrieve data is less than the number of registered data then, the data will be retrieved from the top of registered data list

Since $top query does not retrieve the sorted data, adding the $orderby query is required to retrieve the sorted data.

※will return "400 Bad Request" for a $top query with a non-integer value

Request Query

 

$top={number}

 

Value

Summary

Valid Value

Remarks

{number}

It specify the number of entities to be included in the returned feed

0-10000 numeric characters, (default: 25)

0-100 for specifying $expand query, (default: 25)

 

Sample

·           It will limit to 10 the number of data acquisition list

$top=10

Limitations

·           When considering the performance, used in conjunction with $skip query 
In this case, the recommended value of $top query 50 or less
