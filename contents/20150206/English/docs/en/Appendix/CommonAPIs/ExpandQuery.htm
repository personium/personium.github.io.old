$expand Query

Overview

By specifying the Navigation Property name in $ expand query, retrieving the expanded related information will be also possible.

Up to 100 expanded related information could be retrieved at the time of retrieving the list.

Up to 1000 expanded related information could be retrieved at the time of retrieving one item.

Sorting order of the retrieved expanded information by $expand are as follows:

 

Relevance

Sorting criteria

Order

0 .. 1:0 .. 1

Entity creation date of the navigation property to expand

Descending order

0 .. 1: *

Entity creation date of the navigation property to expand

Descending order

*: 0 .. 1

Entity creation date of the navigation property to expand

Descending order

*: *

Link information creation date with the navigation property to expand

Descending order

 

※ When the Multiplicity is "1", the sorted results will be same as the "0 .. 1"

Request Query

$expand={NavigationPropertyName}

 

Path

Summary

{NavigationPropertyName}

Expand the navigation property name
if you want to specify more than one must be separated by commas

※ If non-existing NavigationProperty name specified in the $expand query, it will return 「400 Bad Request」

 

Specifying the number of NavigationProperty is possible.

     For list retrieving, up to two navigationproperty could be specified.

     For one item retrieving, up to ten navigationproperty could be specified.

 

※ If the specified number of navigation property exits the limit, 「400 Bad Request」 will be returned -bid

Sample

Acquired to expand the information associated with their Animal Keeper

$expand=_Animal

Limitations

·           Only one possible hierarchy deployment-related information

·           In order to show the relevant information was developed to be sure it returns the number of all items in the list to add as related information __count (not supported)
