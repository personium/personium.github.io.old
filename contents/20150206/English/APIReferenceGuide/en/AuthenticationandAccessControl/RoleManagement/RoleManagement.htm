Role Management

Role(s) are defined for each cell, such as administrator role, teacher role, and student role, through which an account (user) can access the particular Cell.

A role can have different set of access privileges defining an account ownership of the cell. The access is based on the concept of RBAC (Role Based Access Control). In RBAC, permission is granted to respective accounts to access the particular Cell(s).

  • Kinds of Roles

There are the following types of Roles:

·        Roles associated with the Box

·        Roles used without string to Box

You must define role(s) to the specified account (user) to access the particular cell. To define the role, you must describe the string definition and body of the four entities: Account, ExtCell, Relation, and ExtRole. Also, you can grant roles to the user entity. 

To define a role, you must associate it with a box. If the role is not associated with the box, it gets automatically associated with the main box. You can access the cell according to the role defined to you. Roles can be defined for the accounts of the cell or for external cell users to access the cell.

You can perform the following activities with the Role:

    1. Create (See Create Role API)

    2. Read/View (See List Role API)

    3. Update (See Update Role API)

    4. Delete (See Delete Role API)

    5. Link role with Account, External Cell, Relation, and External Role.
