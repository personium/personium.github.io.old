Role Class and Role Instance

  • Role Class

A role class points to the role resources, which is defined as the application. It internally has a Role class URL with its token in the transformer cell token. It is used at the time of comparing ExtRole settings.

The Role Class URL structure is as follows:

${SchemaURL}/__role/__/${RoleName}*

 

Where:

SchemaURL: Schema URL to which the specified Role belongs

RoleName: Role name

Sample

https://fqdn/cell1/__role/__/doctor1

  • Role Instance

Role instance provides the actual state of the registered role in a particular cell.

  • Using Role Class and Role Instance URL

When there is a relation between Cells as shown in Figure 1, you can use the following code snippets for the Role Class and Role Instance URL:

appCell

If you want to set the [doctor] of the dataCellA in the [ExtRole] parameter of ExtRole, set the role class URL.

http://fqdn/appCell/__role/__/doctor

 

If you want to set the [doctor] of the dataCellB in the [ExtRole] parameter of ExtRole, set the role instance URL.

http://fqdn/dataCellB/__role/__/doctor

dataCellA

For setting the [doctor] in own cell, set the role instance URL.

http://fqdn/dataCellA/__role/Box_A/doctor

dataCellB

For setting the [doctor] in own cell, set the role instance URL.

http://fqdn/dataCellB/__role/__/doctor
